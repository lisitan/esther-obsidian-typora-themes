# 把想法写得好看，也写得清楚

*Esther Typora Theme · 温暖、克制、有一点手作感*

这是一份用于展示 **Esther Typora Theme** 的 Markdown 案例。它保留了写作工具应有的安静，同时用蓝、黄、红三种颜色建立清晰的阅读节奏。

> **好的排版不是装饰内容，而是帮助读者理解内容。**
>
> 当颜色、字体、间距都有边界，写作者就能把注意力重新交还给文字。

## 01 / 一眼看懂的内容层级

正文使用舒展的行距，适合长时间阅读；标题使用衬线字体，让章节之间有更明确的节奏。你可以自然地使用 **粗体强调**、*斜体注释*、==荧光笔高亮==、[品牌链接](https://github.com/esthersjw/esther-design-system) 和行内代码 `design.tokens`。

### 一份真正能执行的设计清单

- 暖色背景，而不是刺眼的纯白
- 蓝色负责秩序，黄色负责强调，红色只做少量点缀
- 标题与正文混用衬线、无衬线字体，形成编辑感
- 引用、表格、代码块都有完整设计，不使用浏览器默认样式

## 02 / 让复杂信息更容易比较

| 视觉角色 | 品牌色 | 主要用途 |
| :--- | :---: | :--- |
| 主色 | `#2B7FD8` | 标题、链接、结构与秩序 |
| 强调色 | `#F4D758` | 高亮、选择、手绘装饰 |
| 点缀色 | `#E84A5F` | 标记、提醒、少量聚焦 |

### 代码，也可以有温度

```css
:root {
  --blue: #2b7fd8;
  --yellow: #f4d758;
  --red: #e84a5f;
  --paper: #fefcf6;
}

.good-writing {
  color: var(--ink);
  background: var(--paper);
}
```

## 03 / 适合真实写作的细节

1. 用标题建立章节结构，而不是只把字变大。
2. 用列表降低理解成本，而不是堆叠长段落。
3. 用表格呈现比较关系，用代码块呈现技术细节。
4. 用脚注补充信息，让正文保持顺畅。[^theme]
